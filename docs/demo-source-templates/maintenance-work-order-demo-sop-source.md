# Maintenance Work Order Intake Demo SOP Source

Source status: Approved demo seed
Space: Maintenance Work Order Intake
Sensitivity: Low

This is safe demo content for proving the PMI KC KB live Ask path. It contains no real
owner, tenant, address, vendor, ledger, bank, screening, or personally identifying
data.

## Workflow

1. Check the RentVine work order details before giving maintenance guidance.
2. Confirm whether photos, access notes, or tenant details are missing.
3. Use approved request-for-photos wording when the request is incomplete.
4. Escalate vendor assignment when no documented routing rule applies.

## Handling Rules

- Do not choose a vendor or promise dispatch from the KB.
- Do not invent emergency procedures, vendor routing, or photo requirements.
- If the request is missing photos or access details, create a placeholder or use the
  approved request-for-photos template.
- The KB does not update RentVine, Google Chat, vendor records, Gmail, or any system of
  record.

## Escalation

Escalate vendor-selection decisions and undocumented routing rules to Dan or the
process owner. Escalate repeated incomplete-request patterns for SOP review.

## Good Ask Questions For Smoke Testing

- What should the team check when a maintenance request comes in?
- Why should the KB not assign a vendor by itself?
- What should we do when maintenance photos are missing?
